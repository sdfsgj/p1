# Oculai 项目概述

## 最终目标

构建一个 **AI 原生的全流程人才寻聘系统**，以 Claude Code 为编排中枢、PostgreSQL 为全局状态中心，实现从 JD 解析 → 多源搜索 → 语义匹配 → 深度调研 → 评估打分 → 外联策略 → 闭环反馈的端到端自动化，将传统猎头数周的工作量压缩至分钟级。

## 项目做什么

Oculai 是一套异步 8 阶段人才寻聘 Pipeline：

|阶段|名称|做什么|
|-|-|-|
|Phase 0|需求解析|将 JD 文本解析为结构化人才画像（硬技能、学术指标、地域、语言、文化匹配维度）|
|Phase 1|广域搜索|并行对接多源异构数据（学术数据库、代码社区、专业网络），聚合 500–2000 人候选池|
|Phase 2|语义匹配|向量相似度 + 硬性条件过滤，产出 Top 100 高潜候选人及多维度匹配雷达|
|Phase 3|深度调研|抓取论文引用趋势、合作者网络、职业轨迹、技术栈演化，生成可决策的深度档案|
|Phase 4|评估打分|加权多维度评分（学术/工程/领导力/沟通/文化适配/技能/地域/职业阶段）|
|Phase 5|外联策略|基于候选人画像和岗位信息生成个性化触达方案|
|Phase 6|闭环追踪|触达结果回收，知识图谱更新，模型持续优化|

核心设计原则：**Claude Code 做所有决策**（解析 JD、选择搜索策略、评估候选人、生成外联文案）；**Python 工具层只暴露确定性函数**，不调用 LLM API，不做自主决策。

## 使用的开源项目

### 数据库

|项目|用途|
|-|-|
|[PostgreSQL](https://www.postgresql.org/)|主数据库，全局状态中枢|
|[pgvector](https://github.com/pgvector/pgvector)|向量存储与语义相似度搜索|
|[pg\_trgm](https://www.postgresql.org/docs/current/pgtrgm.html)|Trigram 模糊文本搜索|
|[uuid-ossp](https://www.postgresql.org/docs/current/uuid-ossp.html)|UUID 主键生成|

### Python 运行时

|项目|用途|
|-|-|
|[asyncpg](https://github.com/MagicStack/asyncpg)|高性能异步 PostgreSQL 驱动|
|[Pydantic](https://github.com/pydantic/pydantic)|数据校验与配置管理|
|[httpx](https://github.com/encode/httpx)|异步 HTTP 客户端，用于调用外部 API|
|[cachetools](https://github.com/tkem/cachetools)|L1 内存缓存（TTL + LRU）|

### 嵌入与语义匹配

|-|-|

## 整合的外部数据源

* **GitHub REST API** — 仓库搜索 → 贡献者发现 → 用户详情，技术栈映射
* **arXiv OAI-PMH** — 学术论文搜索与作者聚合
* **DBLP Search API** — 计算机科学文献目录检索
* **Semantic Scholar API** — 论文搜索 → 作者详情（h-index、引用量、ORCID、所属机构）
* **OpenAlex API** — 开放学术图谱（论文 → 作者 → 机构 → 研究主题）
* **Google Scholar** — 通过浏览器自动化获取（无官方 API）
